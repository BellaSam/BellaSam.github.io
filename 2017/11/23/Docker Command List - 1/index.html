How to create docker container

How to create image

1. Docker Commit Command

慎用docker commit:

  • 大量无关内容被自动关联
  • 对镜像的操作为黑箱操作，后来人员难以维护
  • 由于分层存储，每一次修改都会让镜像臃肿一次

使用场景:

  • 学习
  • 被入侵后保存现场

2. Dockerfile

Dockerfile 结构 :

  1. FROM : 定义初始镜像，if base image is “scratch”, means it’s empty

  2. RUN 执行命令：两种格式

    • shell 格式，RUN ，类似直接调用shell命令，没执行一次RUN，都创建了一层镜像
    • exec 格式，RUN [‘可执行文件’,’参数1’,’参数2’]，类似调用可执行文件

  3. 构建镜像：在Dockerfile文件目录下执行
    docker build [option]
